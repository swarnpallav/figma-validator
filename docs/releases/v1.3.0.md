## 🚀 v1.3.0 — Visual QA and Fix Guidance

Figma Validator now feels much more like a design drift explainer than a raw mismatch viewer.

---

### ✨ What's new

- Pick Area flow replaces modifier-first selection as the primary interaction
- Mobile Visual QA now opens in a bottom sheet
- Visual QA issue presentation is cleaner and accordion-based
- Dev now gives more direct fix guidance for typography, color, spacing, and shape issues
- Figma plugin now includes a copy prompt flow for agentic implementation workflows

---

### 🧠 Improvements

- Visual QA is more visual-first and less dependent on tree-matching for non-text issues
- Root width and height mismatch are treated more explicitly as geometry truth
- Text mismatches now support clearer diff-style presentation
- Added richer style comparison coverage:
  - font family
  - line height and letter spacing units
  - color
  - border radius
  - stroke width and stroke color
  - opacity and shadow diagnostics in Dev
- Improved repeated-component matching confidence in the geometry matcher

---

### 🐛 Fixes

- Reduced false positives from width-only implementation differences by making width checks optional
- Fixed bogus background-color issues on text nodes
- Fixed bogus blend-mode issues caused by Figma `PASS_THROUGH`
- Fixed bogus letter-spacing issues caused by CSS `normal`
- Removed invalid stroke-style issues caused by comparing Figma stroke alignment with CSS border style
- Improved mobile browser capture so the validator panel does not pollute Visual QA screenshots

---

### 🎯 Why this matters

Instead of only telling you that a region looks off, the tool is now much better at telling you:

→ what changed visually  
→ which property is likely wrong  
→ what value should be changed  

This makes the validator much more useful for MCP-driven UI implementation, where most problems are near-correct CSS drift rather than complete layout failure.

---

### ⚠️ Note

This release improves trust and usability significantly, but Visual QA is still intentionally conservative for non-text mapped issues. Dev remains the richer technical surface for exact fix guidance.
